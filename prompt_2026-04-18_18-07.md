## Что требуеся:

Надо создать скилл разворачивания проекта MCP

Этому скилу указывается файл конфигурации.
Он проверяет, все ли параметры заполнены.
Если что-то не заполнено, то спрашивает, пока все не заполнит в файле конфигурации.
В частности, он должен запросить для config/local.yaml: 
agentTester.openAi.apiKey
agentTester.openAi.baseURL

Если все настройки есть. запускает
fa-mcp <path-to-config.yaml>

Скиллу могут передаваться тектом 
- различные креды и настройки
- требования по созданию нового MCP.
Обычно их достаочно.

Как правило, там будет ссылка на папку или проект, где есть уже тулинг, который надо обернуть в MCP.
Или это может быть указание, какой известный API нужно обернуть в MCP.
Могут быть указания на готовый проект MCP, окуда надо взять полезные фичи.
Тогда Claude Code должен изучить код, на который сослались. Или изучить дополнительную информацию в интернете.

После того, как fa-mcp <path-to-config.yaml> отработает,
Надо сгенерировать для config/local.yaml:
webServer.auth.jwtToken.encryptKey
webServer.auth.permanentServerTokens


Далее нужно реализовать MCP. 
Скилл должен сам себе составить план реализации и тестирования в файле. 
И следовать ему, проставляя галочки в плане реализации.
Самостоятельно провести тестирование. 

Если а процессе анализа выяснится, что не хватает кредов, их надо запросить.
Например, если подразумевается использованеи БД.
А если в сопроводительном тексте сказано, что какие-то фичи не используются, например "не нужно AD и Consul", то тогда и не справшивать про креды для них.

Надо также запускать циклы проверки-доработки инструментария через headless agent tester.
Для упрошения в процессе разработки в config/local.yaml можно установить максимально лояльные значения 
```yaml
agentTester:
  enabled: true
  showFooterLink: true
  useAuth: false

consul:
  service:
    enable: false

webServer:
  auth:
    enabled: false
  adminAuth:
    enabled: false
```
В процессе тестирования через headless agent tester скилл должен сам придумать вопросы агентской системе, использующей тулы,
анализировать ответы и подправлять 
- агентский промпт AGENT_PROMPT
- описание инструментов
- наборы , типы, ограничения и описания тулов
Но достичь удобного и четко работающего ансамбля промпт+тулинг. 

Далее скилл должен создать репозиторий на удаленном сервере. И запушить туда проект.
По умолчанию - это гитлаб сервер.
В состав команд скила должен входить минимальный набор кода для этого.
(подсмотреть можно тут: D:\Work\DOC\ОтделИИ\PLAN\api\report\services\gitlab-service.js и добавить, если чего-то не хватает)
Креды для доступа к гитлабу должны быть предоставлены в тексте сопровождения, а если там нет, то скилл должен их запросить:
- baseUrl (https://gitlab.finam.ru/api/v4)
- token
- group / groupId (если указано имя группы, скилл сам вычисляет groupId по имени группы )
